# Sorbet and Tapioca Runbook

This is the canonical runbook for Sorbet + Tapioca on The Heist. Read it before touching anything under `sorbet/`, before adding or upgrading a gem, and before adding your first `sig` to a new file.

## What's installed

| Gem                | Group            | Purpose                                          |
|--------------------|------------------|--------------------------------------------------|
| `sorbet-runtime`   | runtime (all envs) | `T::Sig`, `T::Struct`, `T::Enum`, runtime checks |
| `sorbet`           | development      | The static checker CLI (`srb`)                   |
| `tapioca`          | development      | RBI generator + DSL compilers                    |

`tapioca` pulls in `sorbet-static` transitively; it's what powers `srb tc`.

### Note on `sorbet/tapioca/require.rb`

The file contains a monkey-patch restoring `has_rest` / `has_keyrest` on `T::Private::Methods::Signature`. Tapioca 0.19.0 still calls those methods, but sorbet-runtime 0.6.13153 removed them. The patch is `# typed: false` and delegates to `rest_name` / `keyrest_name`. Remove it when tapioca ships a fix that matches your pinned sorbet-runtime version (check `bundle exec tapioca gem` without it first â€” if it passes, delete the patch).

## Commands

| Command                                   | When                                                             |
|-------------------------------------------|------------------------------------------------------------------|
| `bundle exec srb tc`                      | Before finishing any change. Required.                           |
| `bundle exec tapioca dsl`                 | After model / association / enum / scope / job changes           |
| `bundle exec tapioca gem`                 | After `bundle install` adds or upgrades gems                     |
| `bundle exec tapioca dsl --verify`        | In CI, and locally before pushing. Fails if DSL RBIs are stale.  |
| `bundle exec tapioca gem --verify`        | In CI. Fails if gem RBIs are stale.                              |

| `bundle exec tapioca require`             | When a constant appears as `todo` â€” regenerates the preload file |

## Strictness ladder

| Sigil                 | Checks                                                              | When                              |
|-----------------------|---------------------------------------------------------------------|-----------------------------------|
| (none â€” default)      | Syntax + constant resolution                                        | Existing files, default           |
| `# typed: false`      | Same as default                                                     | Explicitly opt-out files          |
| `# typed: true`       | Method calls, arg counts, arg/return types when `sig` present       | New service objects / policies / jobs |
| `# typed: strict`     | Every method must have a `sig`; every ivar must be `T.let`-annotated | Hot paths, after broad typing     |
| `# typed: ignore`     | Nothing                                                             | **BANNED.** Breaks downstream files. |

Shopify's rule: "`typed: ignore` means `typed: debt`." Use `# typed: false` if a file can't be typed yet.

## Writing your first sig

```ruby
# typed: true
# frozen_string_literal: true

module HackatimeService
  extend T::Sig

  sig { params(code: String, redirect_uri: String).returns(T.nilable(T::Hash[String, T.untyped])) }
  def self.exchange_code_for_token(code, redirect_uri)
    # ...
  end
end
```

Rules:

- `extend T::Sig` at the top of the class/module.
- `sig { ... }` on the line immediately above the `def`. No blank line.
- `params(arg_name: Type)` for every arg. Kwargs go in the same hash.
- `.returns(Type)` always. Use `.void` for methods that return nothing meaningful.
- `T.nilable(T)` for maybe-nil. Callers must handle nil.
- `T::Hash[K, V]`, `T::Array[V]`, `T::Set[V]` for collections.

## T::Struct and T::Enum

Prefer these over raw Hashes and stringly-typed values.

```ruby
class LeaderboardEntry < T::Struct
  const :user_id, Integer
  const :display_name, String
  const :hours, Float
  const :rank, Integer
end

class ShipStatus < T::Enum
  enums do
    Pending  = new("pending")
    Approved = new("approved")
    Returned = new("returned")
    Rejected = new("rejected")
  end
end
```

For ActiveRecord models, use Rails `enum` instead (`enum :status, { pending: 0, approved: 1, ... }`) â€” Tapioca generates the matching typed methods.

## Escape hatches (and why to avoid them)

| Escape hatch                | What it does                                   | When (if ever)                              |
|-----------------------------|------------------------------------------------|---------------------------------------------|
| `T.unsafe(x)`               | Drops all type info on `x`                     | Never in app code. Only in generated RBIs.  |
| `T.must(x)`                 | Asserts non-nil, raises at runtime if nil      | Only when the invariant is invisible to Sorbet. Prefer `#fetch` or a guard. |
| `T.cast(x, Type)`           | Runtime-checked type assertion                 | Prefer control-flow narrowing via `is_a?`   |
| `T.let(x, Type)`            | Annotates a local or `@ivar`                   | Fine. Required for `@ivars` under `# typed: strict`. |
| `T.bind(self, Type)`        | Annotates `self` inside a proc/block           | Fine in callbacks/closures                  |

If you reach for `T.must` or `T.unsafe`, leave a one-line comment explaining why the type system can't see the invariant.

## Common Sorbet errors

**`Method 'foo' does not exist on 'Bar'`**

- Likely cause: the gem / DSL RBI is stale. Run `bundle exec tapioca dsl` or `bundle exec tapioca gem` and re-check.
- If it's a real typo, fix it.

**`Expected 'String' but found 'T.nilable(String)'`**

- A method returned `nil` where a non-nil was expected. Guard with `return unless x`, use `#fetch`, or narrow the sig to `T.nilable(...)` and handle nil at the call site.

**`Method call might be unsafe`**

- Calling a method on a `T.nilable` without narrowing. Fix: guard or narrow.

**`Expected 'X' but found 'Y' for argument 'z'`**

- The caller passed the wrong type. Fix the caller. Don't widen the sig to make it work.

**`Type T.class_of(X) does not match supertype`**

- Usually seen on `ActiveSupport::Concern`. The concern assumes a host class Sorbet can't see. Either mark the concern `# typed: false` or use `requires_ancestor` inside an `interface!` module.

## CI integration

`bundle exec srb tc` and `bundle exec tapioca dsl --verify` should run in CI on every PR. Treat failures as blocking. Stale DSL RBIs usually mean someone forgot to commit the regenerated file after a model change â€” fix locally and push.

## Adoption ladder (Shopify pattern)

1. **Baseline**: all existing files implicit `# typed: false`. Done. `srb tc` clean.
2. **New code at `# typed: true`**: every new service, policy, job, form object, Data class.
3. **Touch upgrades**: when you edit an existing file substantively, promote it to `# typed: true` in the same PR.
4. **Ratchet to 100% `# typed: true`**: target state for app code.
5. **Hot paths at `# typed: strict`**: only when the codebase is broadly typed and the file's on a critical path.

Do not do a big-bang upgrade. Shopify took several years to ratchet 75,000 files. At The Heist's scale this is weeks of incremental work, not a single PR.

## When Tapioca and sorbet-runtime versions drift

Tapioca calls internal methods on sorbet-runtime's `T::Private::Methods::Signature`. When these APIs change, `tapioca gem` or `tapioca dsl` breaks with a `NoMethodError`. Known incidents:

- **`has_rest` / `has_keyrest` removed** (sorbet-runtime ~2024). Tapioca 0.19.0 still calls them. Workaround: see `sorbet/tapioca/require.rb`.

When a future drift happens:

1. Read the full stack trace. The method name is the clue.
2. Check sorbet-runtime's current `Signature` methods: `bundle exec ruby -e 'require "sorbet-runtime"; pp T::Private::Methods::Signature.instance_methods(false).sort'`.
3. If the method was renamed, add a small alias in `sorbet/tapioca/require.rb` as a monkey-patch.
4. If the method was removed entirely, pin tapioca or sorbet-runtime to a compatible version and leave a comment in the Gemfile.

## When srb tc complains but you're right

Sometimes Sorbet is wrong (gem RBIs are incomplete, or a DSL isn't fully typed). Options, in order of preference:

1. **Fix the RBI**: `bundle exec tapioca dsl --only=MyModel` or edit the DSL compiler if it's a custom Rails extension.
2. **Write a local RBI override**: put it in `sorbet/rbi/shims/` (create the directory if needed; not auto-generated, checked into git). Keep shims small and well-commented.
3. **Narrow with `T.let`** at the call site.
4. **`T.cast`** as a last resort.
5. **Never `T.unsafe`** in app code.

## Things that used to work but now go through type checking

- Returning a Hash from a service method â†’ now must be `T::Hash[String, T.untyped]` or better.
- Passing `nil` to a `String`-typed arg â†’ compile error; change the sig or fix the caller.
- `Model.find_by(id: x).name` â†’ `find_by` returns `T.nilable(Model)`; you must guard.

These aren't regressions â€” they're bugs Sorbet now catches before they hit production. Fix at the source.
